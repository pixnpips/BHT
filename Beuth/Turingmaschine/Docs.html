Turing Maschine Dokumentation

Hilfe/ Funktion

Dem Nutzer werden folgende Funktionen zu Verfügung gestellt:

Direktes Eingeben eines eigenen Embedded-Reeber-Wortes in das Input Feld

Check Input:
Vorab-Prüfung des Inputs, ob alle Zeichen im Eingabealphabet enthalten sind

Generate reandom e-Reeber/ Generate wrong e-Reeber:
automatisches Erstellen von zufälligen richtigen und falschen Embedded Reeber Worten mittels von Generatorfunktionen
Ein falsches Wort kann dabei Fehler im inneren Reeber Bereich haben, als auch im äußeren damit auch
die Turing Maschinen spezifische Funktion geprüft werden kann

Sequence/Step:
Starten der graphisch simulierten Überprüfung des im Input eingegebenen Embedded-Reeber-Wortes

Reset:
Zurücksetzen der Turingmaschine in den Ausgangszustand/ Unterbrechen der Sequenz

Animation Speed:
die Animationsgeschwindigkeit kann jederzeit verändert werden

Technik

Erstellung:
html, css, js, svg, png

Librarys/ Frameworks:

JQuery: Interaktion zwischen den HTML Dokumenten, vereinfachte Auswahl und Zugriff auf CSS Eigenschaften der HTML Elemente.
Erstellen der Animationen

Bootstrap: Allgemeines CSS Layout, customizen der Elemente über das CSS-main-Sheet

Implementierung

Der Input wird in ein Array vom Typ String gespeichert, das im Verlauf der Animation bearbeitet wird
Zum erstellen der e-Reeber.Wörter wurde eine rekursive Gruppe von Funktionen benutzt.

Die Vorab-Prüfung der Eingabe gleicht den Input mit einem konstanten Array ab, um zu ermitteln, ob alle Zeichen Teil
des Eingabealphabetes sind

Die Zeichen des Arrays werden in der Simulation Zustandsweise überprüft. Jeder Zustand besitzt seine eigene Funktion,
alle Zuständsfunktionen sind rekursiv implementiert, sodass sie sich gegenseitig aufrufen.
Die aktuellen Zustände und auch die Übergänge werden farblich markiert.
Anschließend wird das verarbeitete Zeichen durch ein Blank ersetzt. Es kommt weiterhin eine Function Map,
die Timing Funktion SetTimeout und diverse Boolean Variablen zum Einsatz

Der Zustand des Eingabebandes wir mittels JQuery animate implementiert.

Das Feedback erhält der User über Farbsemantik und einen Info Text Bereich in der Web App.
