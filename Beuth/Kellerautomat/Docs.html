Implementierung

Die möglichen Ausdrucke für jede einzelne Produktionsvorschrift wurden in Arrays vom Typ String gespeichert.
Die Grammatik aus der vorherigen ESA wurde zum Erstellen der mathematischen Audrücke verwendet, zusätzlich hat der User
die Möglichkeit eigene Ausdrücke einzugeben. Die Eingaben und die generierten Ausdrücke müssen in den Kellerautomat
eingelesen werden. Die Eingabe wird in Arrays umgewandelt und deren Zeichen werden stellenweise mit konstanten Arrays aus dem Code abgeglichen
um korrekte Zustandsübergänge zu gewärleisten. Die aktuellen Zustände und auch die Übergänge werden farblich markiert.
Anschließend wird das verarbeitete Zeichen gelöscht und ggfs. ein Zeichen in das KellerStack geschrieben. Es kommt eine Function Map,
die Timing Funktion SetInterval und diverse Boolean Variablen.n
Es wird dem Benutzer folgende Funktionalität zur Verfügung gestellt: Erstellen eines korrekten Ausdruckses,
Erstellen eines falschen Ausdruckes, Eine Step-Through Funktion und eine Sequenz, die korrekte Wörter
akzeptiert oder ablehnt. Das Feedback erhält der User über Farbsemantik und einen Info Text Bereich in der Web App.
Der User hat die Möglichkeit beliebig zwischen Sequenz und Step zu wechseln, als auch die Sequenz zu pausieren und wieder aufzunehmen.
