Implementierung

Die möglichen Ausdrucke für jede einzelne Produktionsvorschrift wurden in Arrays vom Typ String gespeichert.
Die Hauptausgabe des aktuellen Schrittes ergibt sich aus einer globalen Variablen vom Typ String, die Anhand der Produktionsvorschriften
mit replace-Methoden aktualisiert wird. Die Anwendung der Methoden erfolgt teilweise über Loops, die über das von der jeweiligen
Vor-Produktionsregel erstellte Wort iterieren und dieses wieder in die globale Variable zurückschreiben.
Der Zugriff auf die Arrays erfolgt über das generieren einer Random.Zahl, die auf die Indexe des jeweiligen Ausdrucks-Arrays zugreift.
Auf die Ausgabe jedes einzelnen Schrittes wird bewusst verzichtet, da insgesamt nur 5 Produktionsvorschriften aktiv ausgeführt werden
Es wird dem Benutzer folgende Funktionalität zur Verfügung gestellt: Erstellen eines korrekten Ausdruckses,
Erstellen eines falschen Ausdruckes, Eine Step-Through Funktion und eine Sequenz, die korrekte Wörter und deren
Generierung visualisieren. Das Feedback erhält der User über Farbsemantik und einen Info Bereich in der Web App.
